Partie HTML :
Au début du cours j'avais quelques connaissances en html mais, je n'utilisais que des div pour séparer les éléments.
J'avais dejà appris dans mon entreprise à respecter les règles de google pour le SEO.

Pour ce projet, j'ai fait plusieurs choix :
    - Utilisation de balise HTML 5.
    - Utilisation de meta pour du responsive.
    - Importation de font depuis le HTML et non depuis le css pour gagner de l'optimisation côté navigateur.
    - Quelques éléments html écrits en dur (pas générés par js) pour que du contenu sois affiché sans JS.
    - Positionnement des balises scripts en bas de page pour optimiser le premier affichage (SEO)

/////////////////////////////////////////////////////////

Partie CSS :
Au début du cours, je connaissais déjà les flexbox et les bonnes pratiques pour du responsive.
Je connais également bien le framework bootstrap.

Pour ce projet, j'ai fait plusieurs choix :
    - Utilisation de css3 natif.
    - Utilisation des flexbox.
    - Utilisation des keyFrame pour des animations.
    - Utilisation des classes contenant des transform pour des animations optimisées.
    - Utilisation des mediaQuery pour rendre le menu et l'interface responsive.

/////////////////////////////////////////////////////////

Partie JS/JSON :
Au début du cours, j'avais déjà quelques connaissances en javascript natif.
J'avais déjà réalisé quelques sites en javascript natif et aussi un système de configuration sur mesure pour du e-commerce.
Je sais déjà utiliser des framework comme react et vuejs.

Pour ce projet, j'ai fait plusieurs choix :
    - Utilisation de la méthode fetch pour les appels ajax.
    - Utilisation des hash pour créer un système de navigation avec un routeur.
    - Utilisation des cookies pour l'authentification.
    - Utilisation de la lib externe charJs pour générer des graphiques.
    - Preconfiguration de chaque requette pour un fonctionnement avec un vrai back-end.

Synthèse personnelle:

Je suis content du rendu final.
L'objectif était de construire une application web qui comprenait :
    - Une compatibilité avec tous les supports
    - Une interface personnalisable
    - Des animations fluides et utile (elles permettent de cacher des temps de chargement)
    - Une SPA pour améliorer l'expèrience utilisateur.
    - La possibilité d'ajouter des modules (pages) sans modifier la base du code

Les améliorations possibles sont :
    - Ajout d'un système d'extension, le système est prévu grace au MVC en FRONT.
    - Couleurs personnalisables.
    - Ajout de service worker pour créer une PWA.
    - Ajout de notifications push pour prévenir des alertes.

Gràce à ce projet, j'ai appris de nombreuses choses :
    - J'ai appris à utiliser la lib chartJs car je ne la connaissais pas.
    - J'ai appris à utiliser les bonnes balises HTML5.
    - J'ai appris à créer un système de routage en JS.
    - J'ai appris à utiliser les cookies en JS.
